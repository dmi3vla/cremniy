## Phase 4 — Autonomous project-level Canvas

**Problem**: CanvasTab lives inside FileTab → bound to a specific `.cpp` file. Codemap is project-level.

**Solution**: Add `FilesTabWidget::openCodemapTab()` that creates a special FileTab with `filePath=projectRoot` and `tabTitle="Codemap"`.

### Files to modify

| File | Change |
|------|--------|
| `src/ui/filestabwidget.h` | Add `openCodemapTab()` method |
| `src/ui/filestabwidget.cpp` | Implement: create FileTab with projectRoot as path, set tab title "Codemap" |
| `src/app/IDEWindow/idewindow.h` | Add `openOrCreateCodemapCanvas()` |
| `src/app/IDEWindow/idewindow.cpp` | Implement: use `openCodemapTab()` instead of `openFile(cpp)`, switch to Canvas tool tab, call `enterSemanticMode()` |
| `src/ToolTabs/Canvas/canvastab.cpp` | `setFile()` → if path is directory (not file), skip Gource/animator init, just set projectPath |

### Implementation

1. **`FilesTabWidget::openCodemapTab(projectPath)`**:
   - Create `FileTab(this, projectPath)` — ToolsTabWidget gets created with `path=projectRoot`
   - Set tab title to "Codemap"
   - Switch to this tab
   - Return the FileTab

2. **`IDEWindow::openOrCreateCodemapCanvas()`**:
   - Find existing CanvasTab via `canvasTab()`
   - If not found → `m_filesTabWidget->openCodemapTab(projectPath())`
   - Find the CanvasTab in the new tab
   - Switch to Canvas tool tab (setCurrentIndex for ToolsTabWidget)
   - Call `canvas->setProjectRoot(projectPath())`
   - Call `canvas->enterSemanticMode()`

3. **`openOrGenerateConceptMap()`**: replace `openOrCreateCanvasTab()` with `openOrCreateCodemapCanvas()`

4. **`CanvasTab::setFile()`**: if `QFileInfo(filepath).isDir()`, just set `m_projectPath = filepath`, skip parser/animator creation
